# 道具系统规格（代码基准）

## 1. 道具总览

| Key | 道具 | 获取点 | 关键用途 |
|---|---|---|---|
| `key` | 钥匙 | S5 邮箱 | S4 开大门 |
| `wateringCan` | 水壶 | S8 橱柜 | S8 接水、S7 浇花 |
| `wood` | 木头 | S10 照片弹窗 | S16 修沙发 |
| `saw` | 锯子 | S12 鱼点亮后 | S16 修沙发 |
| `hammer` | 锤子 | S15 抽屉二次点击 | S16 修沙发 |
| `nail` | 钉子 | S15 抽屉二次点击 | S16 修沙发 |

## 2. 背包显示与选中

实现：`InventoryDisplay`

- 所有道具图标在场景创建时初始化，默认隐藏
- `refresh()` 按 `GameState.inventory` 控制显示
- 选中逻辑：
  - 点击道具 -> 设为选中
  - 再点同一道具 -> 取消选中
- 选中贴图：
  - 木头/锯子/锤子/钉子统一使用 `22/物品/*选择.png`

## 3. 道具流程（逐项）

### 3.1 钥匙 `key`

- 获取：
  - S5 打开邮箱 -> 点击钥匙
- 使用：
  - S4 选中钥匙点击门
  - 开门后消耗钥匙（`consumeItem('key')`）

### 3.2 水壶 `wateringCan`

- 获取：
  - S8 开柜拿壶
  - 获取后不自动选中
- 接水：
  - 必须选中水壶
  - 必须点击 `10/开柜门壶在柜子里.png` 指定区域
  - 完成后 `waterFilled=true`
- 浇花消耗：
  - S7 浇花后水壶从背包移除
  - 相关状态一起清理：
    - `wateringCanSelected=false`
    - `waterFilled=false`

### 3.3 木头 `wood`

- 获取：
  - S10 点亮花云 -> 相机弹窗 -> 点击木头
- 点击木头后规则：
  - 入包（`wood=true`, `woodAcquired=true`）
  - 木头立即从当前弹窗消失
  - 弹窗仍需手动关闭

### 3.4 锯子 `saw`

- 获取：
  - S12 鱼全亮后出现锯子
  - 点击后入包并触发池塘点亮

### 3.5 锤子 `hammer` + 钉子 `nail`

- 获取：
  - S15 密码 `2513`
  - 延时约2秒进入开抽屉
  - 第一次点抽屉弹日记
  - 第二次点抽屉同时获取锤子和钉子

## 4. 修沙发道具消耗与状态机

S16 当前实现并非真实拖拽吸附，而是“选中道具 + 点击推进状态”：

1. 木头（sofa点击）
2. 锯子（屏幕点击）
3. 切开（屏幕点击）
4. 留下木料（屏幕点击）
5. 木料吸附（屏幕点击）
6. 钉子（屏幕点击）
7. 锤子（屏幕点击）
8. 锤击一次（屏幕点击）
9. 锤击二次（屏幕点击，完成）

## 5. 与场景门禁的关系

- 卧室门禁依赖花园/池塘完成度，不直接依赖道具。
- 但要推进到修沙发，实际上必须收齐：
  - `wood`（S10）
  - `saw`（S12）
  - `hammer` + `nail`（S15）

## 6. 注意事项（代码事实）

1. 水壶是一次性流程道具，浇花后移除。
2. 木头点击后“立即从照片消失”是当前生效规则。
3. 修沙发使用的是状态推进点击，不是完整拖拽系统。
