# 交接文档：启动与运维说明

本文用于新同学接手后快速启动、验证与停服。以下步骤以当前代码结构为准。

## 1. 目录结构（关键）

- 项目根目录：`项目三素材 - 副本`
- 前端代码目录：`code`
- 入口页面：`code/index.html`
- 资产清单：`code/assets/manifest.json`
- 规则归档：`doc/归档-代码基线-v0.3.27.md`

## 2. 环境要求

- 操作系统：macOS / Linux / Windows 均可
- 运行方式：静态文件服务（无需后端 API）
- 建议工具：`python3`（用于快速启动 http server）

## 3. 启动方式

在终端执行：

```bash
cd "code"
python3 -m http.server 8000
```

启动成功后访问：

- 本机地址：`http://127.0.0.1:8000`

> 当前交接时刻服务默认处于关闭状态，需要手动执行上述命令启动。

## 4. 常见验证点（接手后先测）

1. 进入客厅 C，拿水壶后不会自动选中，需手动选中才能接水  
2. 浇花后花园门立即可进（不需切场景刷新）  
3. 花园仰望点击木头后，木头入包且在弹窗中立即消失  
4. 卧室门在未完成点亮前不可进入  
5. 密码 `2513` 后存在约2秒停顿再开抽屉  
6. 沙发修复中“留下木料 -> 吸附木料”为两步点击  
7. 修好沙发后关闭照片弹窗仍在客厅，笔记图标变为打开态  
8. 文字输入阶段有闪烁光标，提交后文字保留、按钮替换为章；再点击显示总结图
9. MainMenu/Intro 缓慢淡入；UI 点击有音效；HouseExterior 起切换后段 BGM
10. Mailbox 开邮箱弹 `6/弹窗.png`，X 关闭

## 5. 调试方式

- 游戏内按 `D`：显示/隐藏调试面板
- 调试面板可看：
  - 当前场景
  - 纹理加载情况
  - 背包状态
  - 关键标志位

## 6. 版本与缓存

- 当前版本号定义在：`code/src/main.js` 的 `APP_VERSION`
- 页面脚本缓存参数在：`code/index.html`（`?v=...`）
- 每次发布或交付建议同时更新：
  1. `APP_VERSION`
  2. `index.html` 所有脚本版本参数

## 6.1 快速确认版本

浏览器打开后，左上角会显示版本徽标（由 `main.js` 注入），应与 `APP_VERSION` 一致。

## 7. 停止服务

若当前终端正在前台运行服务，按：

- `Ctrl + C`

若服务在后台（占用 `8000`），可执行：

```bash
lsof -ti tcp:8000 | xargs kill
```

## 9. Vercel 一键发布（最简）

已在项目根目录提供 `vercel.json`，用于把根路径请求重写到 `code/` 目录，无需改现有前端结构。

### 9.1 配置文件

- 文件：`vercel.json`
- 作用：
  - `/` -> `code/index.html`
  - 其余静态资源路径（`/src/*`、`/assets/*`）-> `code/*`

### 9.2 一键发布步骤

1. 将当前项目推到 GitHub（或 GitLab/Bitbucket）。
2. 打开 [Vercel](https://vercel.com/) 并导入该仓库。
3. Framework Preset 选 `Other`（或保持自动识别）。
4. Build Command 留空（静态站点无需构建）。
5. Output Directory 留空（由 `vercel.json` 重写处理）。
6. 点击 `Deploy`。

### 9.3 发布后检查

- 首页能打开并进入主菜单。
- 浏览器 Network 中 `src/*.js`、`assets/*` 均返回 200。
- 游戏内左上角版本号与 `code/src/main.js` 的 `APP_VERSION` 一致。
