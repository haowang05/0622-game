# 规则与流程归档（代码基线）

本归档以当前代码为唯一准绳，版本：`v0.3.28-20260630-1200`。
若其他文档出现冲突，以本文件与代码状态机为准。

## 1. 基础运行与框架

- 引擎：`Phaser 3`
- 画布：`2800 x 1840`
- 入口：`code/index.html`
- 启动文件：`code/src/main.js`
- 场景切换统一使用 `QW.TransitionManager`（跨场景黑色淡入淡出；同场景 restart 不 fade）
- 音频统一使用 `QW.AudioManager`（BGM + SFX + 点击冲突规则）
- 全局状态统一使用 `QW.GameState`
- 弹窗统一使用 `QW.PopupScene` + `QW.showPhotoPopup / QW.closePhotoPopup`

### 1.1 场景注册顺序（main.js）

`Boot, AssetLoader, MainMenu, Intro, Office, Dialog, HouseExterior, Mailbox, LivingRoomA, LivingRoomB, LivingRoomC, GardenEntry, GardenSky, PondApproach, PondFish, LivingRoomReturn, Bedroom, Password, SofaRepair, Ending, PopupScene`

## 2. 全局规则（按代码）

- 道具库存：`key` / `wateringCan` / `wood` / `saw` / `hammer` / `nail`
- 卧室门禁：必须满足 `allGardenLit() && pondLitUp`
- 水壶流程：
  - `LivingRoomC` 拿壶后不自动选中
  - 必须选中后点击指定区域才可接水
  - 浇花后水壶被消耗并从背包移除
- 花园点亮：
  - 花 5 + 云 3 任意顺序点亮
  - 点亮过程采用无缝替换（不整场景重启）
- S10 木头规则：
  - 木头仅在照片弹窗内出现
  - 点击木头加入背包后，木头立即从当前弹窗消失
  - 弹窗不会因点木头自动关闭
- 池塘规则：
  - 池塘入口开启条件：`woodAcquired || allGardenLit`
  - 鱼全部点亮后出现锯子
  - 拿锯子后池塘点亮
- 沙发修复流程：
  - 木头、锯子、木料残留、木料吸附、钉子、锤子、敲击状态由 `GameState.flags` 驱动
  - “留下的木料”与“吸附到椅子腿”是两次独立点击步骤

## 3. 主流程（按场景）

1. `MainMenu` -> `Intro` -> `Office` -> `Dialog` -> `HouseExterior` -> `Mailbox`
2. 客厅循环：
   - 暗态：`LivingRoomA` ↔ `LivingRoomB` ↔ `LivingRoomC`
   - 点亮后：`LivingRoomA` ↔ `LivingRoomB` ↔ `LivingRoomReturn`（`LivingRoomB` 仍可进入）
3. 花园：
   - `GardenEntry`（草坪去仰望、池塘入口）
   - `GardenSky`（点花云、拍照拿木头）
   - `PondFish`（点鱼拿锯子）
4. 卧室链：
   - `Bedroom` -> `Password`
   - 密码默认 `0000`，输入 `2513` 后延迟约2秒开抽屉
   - 第一次抽屉：日记弹窗；第二次：拿锤子+钉子
5. 修沙发：
   - `LivingRoomA` 点亮态收齐四道具后，先提示再进 `SofaRepair`
   - 修好后关闭弹窗回 `LivingRoomA`

## 4. 修好沙发后的结局链路（当前实现）

当前代码以 `LivingRoomA` 内覆盖链路为主，不强制跳 `Ending` 场景：

- 修好后 `LivingRoomA` 出现相机
- 点击相机弹出 `22/修好沙发照片弹窗.png`
- 关闭照片弹窗后，仍在客厅，笔记图标切换为 `打开的本子图标.png`
- 点击打开本子后进入覆盖序列：
  - `内容.png`
  - 点击 -> `盖章.png`
  - 点击 -> `跳转文本框提示.png`
  - 点击 -> `文本框背景.png`
  - 点击 -> `文本框输入.png` + `文本框√.png`
  - 键盘输入支持闪烁光标
  - 点击提交后保留文字，仅按钮替换为 `文本框章.png`
  - 再点击 -> `总结.png` + `gameCompleted=true`（章阶段不可结束）

> 备注：`EndingScene` 仍在工程中并可独立运行，但当前主链路已经在 `LivingRoomA` 内完成结局覆盖流程。

## 7. 验收建议（最小回归）

- S8 拿壶 -> 手动选壶 -> 指定区域接水 -> S7 浇花后即时开门
- S10 点木头后木头即时从弹窗消失，弹窗不自动关
- 卧室门在未点亮完成前不可进
- `2513` 输入后有2秒停顿
- S16 木料“残留 -> 吸附”为两次点击
- 修好后相机弹窗关闭不黑屏，仍在客厅
- 输入阶段可见闪烁光标，提交后文字保留
