# 游戏概念（实现对齐版）

## 1. 项目定位

- 类型：点击交互解谜（Point-and-Click）
- 平台：Web（浏览器）
- 引擎：Phaser 3
- 画布：`2800x1840`
- 文本策略：剧情文案以 PNG 形式呈现，不做 DOM 文本剧情系统

## 2. 叙事主旨

主旨仍是“修复与和解”：  
通过从“外部任务（修沙发）”进入“内在感受（结局输入与总结）”的结构，让玩家从工具驱动过渡到情绪收束。

## 3. 当前实现下的叙事骨架

1. 接受委托（办公室链）
2. 进入住所并探索客厅
3. 完成花园/池塘点亮，环境从冷到暖
4. 在卧室/密码抽屉补全修复工具
5. 完成沙发修复
6. 在客厅内触发结局覆盖链（输入、提交、总结）

## 4. 视觉叙事机制（代码事实）

- 点亮机制采用“直接切图”，避免场景闪烁重启感
- 弹窗统一交给 `PopupScene`，保证层级清晰
- 关键情绪节点：
  - 花园亮起
  - 池塘亮起
  - 沙发修好
  - 结局覆盖链（黑底+内容/盖章/输入/总结）

## 5. 关键系统边界

- 有两个“结局实现入口”：
  1. `LivingRoomAScene` 内覆盖链（主流程当前使用）
  2. `EndingScene` 独立场景（保留）
- 文档与讨论若出现分歧，以当前代码入口（1）为主。

## 6. 设计约束（对后续迭代）

1. 新需求若改流程，先更新 `doc/归档-代码基线-v0.3.27.md`。
2. 不允许在关键状态上出现“文档说 A、代码做 B”。
3. 与玩家感知强相关的交互（门禁、弹窗关闭、道具消失时机）必须精确描述并回归测试。
